# 搜索

## 概述

EasyDoc 内置了一个强大的全文搜索功能，完全在客户端运行。无需维护搜索服务器，也无需配置任何 API —— 搜索索引在构建时生成，并嵌入到你的静态站点中。用户可以即时获得所有文档页面的模糊匹配结果。

## 打开搜索

在文档的任何位置按下 **Cmd+K**（macOS）或 **Ctrl+K**（Windows/Linux）即可打开搜索对话框。你也可以点击顶部导航栏中的搜索按钮。

搜索对话框提供简洁、友好的键盘操作界面：

- 开始输入即可看到即时结果
- 使用 **↑** 和 **↓** 方向键浏览结果
- 按 **Enter** 打开选中的结果
- 按 **Esc** 或点击对话框外部即可关闭

## 搜索原理

### 模糊匹配

EasyDoc 使用模糊搜索，这意味着你不需要输入完全精确的词语。部分匹配、拼写错误和近似查询都能正常工作：

- 搜索 "instal" 会匹配到 "Installation"
- 搜索 "confg" 会匹配到 "Configuration"
- 搜索 "mrkdown" 会匹配到 "Markdown"

### 搜索结果分组

搜索结果按文档分组，方便查看每个匹配项来自哪个页面。每条结果展示：

- 文档标题
- 包含匹配文本高亮的相关摘要
- 匹配项所在的章节标题

## 搜索索引

搜索索引在构建时生成，通过扫描 `docs/` 目录下的所有 Markdown 文件。Vite 插件会提取：

- Frontmatter 中的文档标题
- 所有标题文本（h1 至 h6）
- 正文内容（纯文本，去除 Markdown 格式）

索引会输出为 `dist/search-index.json`。在开发过程中，它也会通过 `/search-index.json` 提供服务；在生产环境中，客户端会在首次使用时获取该文件，并在整个会话期间缓存。

你无需进行任何配置 —— 只要添加新的 Markdown 文件，它就会在下一次构建时被自动索引。

## 使用技巧

- **搜索不区分大小写** —— `INSTALLATION`、`installation` 和 `Installation` 都会返回相同的结果。
- **结果实时更新** —— 随着你的输入，结果列表会即时更新，无需按 Enter 键来触发搜索。
- **所有语言都会被索引** —— 如果你有多语言的文档，搜索功能在所有语言中都能正常工作。
- **当前语言的结果优先显示** —— 搜索会索引所有配置的语言。当前语言的结果会优先显示。

## 技术细节

- 基于 **Fuse.js v7**，一个轻量级的模糊搜索库。
- 索引由文档标题、章节标题和正文内容（已去除 Markdown 语法）构建而成。
- 搜索索引 JSON 在首次使用时获取一次，并在整个会话期间缓存，因此在初始页面加载后，即使离线也能使用搜索功能。
